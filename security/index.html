Issues Found

1. XSS

This is attack is quite dangerous because it can lead to invalid data for third party users as well as allow unauthorized people to manipulate the product.

Using the /sendLoactions API one can pass whatever they want into a field, for instance script tags with valid JS, and then when that is accessed later it will run the javascript.

This can be resolved by escaping user input before storing it so that potentially dangerous characters can be escaped to their HTML components. This way they will still be displayed correctly when accessed but not mistaken for javascript or something else nasty.
